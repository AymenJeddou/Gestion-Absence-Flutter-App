Base de données MySQL 
À créer dans phpMyAdmin (XAMPP) 
 
Script SQL fourni 
Créez une base de données nommée gest_absence et exécutez le script suivant dans 
phpMyAdmin : 
 
-- ============================================ 
--  Base de données : gest_absence 
--  Créer dans phpMyAdmin (XAMPP) 
-- ============================================ 
  
CREATE DATABASE IF NOT EXISTS gest_absence 
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; 
USE gest_absence; 
  
-- Table des utilisateurs (commun aux 3 rôles) 
CREATE TABLE utilisateurs ( 
  id         INT AUTO_INCREMENT PRIMARY KEY, 
  nom        VARCHAR(100) NOT NULL, 
  prenom     VARCHAR(100) NOT NULL, 
  email      VARCHAR(150) NOT NULL UNIQUE, 
  password   VARCHAR(255) NOT NULL, 
  role       ENUM('admin','enseignant','etudiant') NOT NULL, 
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP 
); 
  
-- Table des classes 
CREATE TABLE classes ( 
  id     INT AUTO_INCREMENT PRIMARY KEY, 
  nom    VARCHAR(50) NOT NULL, 
  niveau VARCHAR(50) 
); 
  
-- Table des étudiants (liée à utilisateurs + classes) 
CREATE TABLE etudiants ( 
  id             INT AUTO_INCREMENT PRIMARY KEY, 
  utilisateur_id INT NOT NULL, 
  classe_id      INT NOT NULL, 
  FOREIGN KEY (utilisateur_id) REFERENCES utilisateurs(id) ON DELETE CASCADE, 
  FOREIGN KEY (classe_id)      REFERENCES classes(id) 
); 
  
-- Table des enseignants 
CREATE TABLE enseignants ( 
  id             INT AUTO_INCREMENT PRIMARY KEY, 
  utilisateur_id INT NOT NULL, 
  specialite     VARCHAR(100), 
  FOREIGN KEY (utilisateur_id) REFERENCES utilisateurs(id) ON DELETE CASCADE 
); 
  
-- Table des matières 
CREATE TABLE matieres ( 
  id  INT AUTO_INCREMENT PRIMARY KEY, 
  nom VARCHAR(100) NOT NULL 
);

-- Table des séances 
CREATE TABLE seances ( 
  id            INT AUTO_INCREMENT PRIMARY KEY, 
  enseignant_id INT NOT NULL, 
  classe_id     INT NOT NULL, 
  matiere_id    INT NOT NULL, 
  date_seance   DATE NOT NULL, 
  heure_debut   TIME NOT NULL, 
  heure_fin     TIME NOT NULL, 
  FOREIGN KEY (enseignant_id) REFERENCES enseignants(id), 
  FOREIGN KEY (classe_id)     REFERENCES classes(id), 
  FOREIGN KEY (matiere_id)    REFERENCES matieres(id) 
); 
  
-- Table des absences 
CREATE TABLE absences ( 
  id          INT AUTO_INCREMENT PRIMARY KEY, 
  seance_id   INT NOT NULL, 
  etudiant_id INT NOT NULL, 
  statut      ENUM('present','absent') NOT NULL DEFAULT 'present', 
  UNIQUE KEY unique_appel (seance_id, etudiant_id), 
  FOREIGN KEY (seance_id)   REFERENCES seances(id), 
  FOREIGN KEY (etudiant_id) REFERENCES etudiants(id) 
);

Structure de l'API PHP 
Fichiers à créer dans htdocs/gest_absence_api/ 
 
Arborescence recommandée 
 
htdocs/ 
└── gest_absence_api/ 
    ├── config/ 
    │   └── database.php        ← connexion MySQL 
    ├── auth/ 
    │   └── login.php           ← POST /auth/login 
    ├── admin/ 
    │   ├── etudiants.php       ← GET / POST / PUT 
    │   ├── enseignants.php     ← GET / POST / PUT 
    │   ├── classes.php         ← GET / POST 
    │   └── seances.php         ← GET / POST (affecter) 
    ├── enseignant/ 
    │   ├── seances.php         ← GET (ses séances) 
    │   └── absences.php        ← POST (soumettre appel) 
    └── etudiant/ 
        ├── profil.php          ← GET (son profil) 
        └── absences.php        ← GET (ses absences) 
 
 
 
Format JSON attendu : Chaque endpoint retourne toujours un objet JSON. En cas de 
succès : {"success":1, "data":[...]}. En cas d'erreur : {"success":0, "message":"..."}.

Écran de connexion — comportement attendu 
 
login_screen.dart 
• 
Deux champs : Email et Mot de passe (obscureText) 
• 
Bouton « Se connecter » appelle POST /auth/login.php 
• 
En cas de succès, sauvegarde l'id et le rôle avec shared_preferences 
• 
Redirige automatiquement selon le rôle : 
– role == 'admin'       → AdminHome 
– role == 'enseignant'  → EnseignantHome 
– role == 'etudiant'    → EtudiantHome 
• 
En cas d'erreur, affiche un SnackBar avec le message de l'API

Détail des interfaces par rôle 
 
Interface ADMIN 
admin_home.dart 
• 
BottomNavigationBar ou Drawer avec 4 sections : Étudiants, Enseignants, Classes, 
Séances 
• 
Chaque section affiche une liste chargée depuis l'API (FutureBuilder + 
ListView.builder) 
 
etudiants_screen.dart 
• 
Récupère GET /admin/etudiants.php → affiche nom, prénom, classe 
• 
Bouton + → formulaire d'ajout (nom, prénom, email, mot de passe, sélection de 
classe) 
• 
Tap sur un étudiant → formulaire de modification (PUT) 
 
enseignants_screen.dart 
• 
Même logique que etudiants_screen, avec le champ spécialité 
 
classes_screen.dart 
• 
Liste des classes + formulaire d'ajout (nom, niveau) 
 
seances_screen.dart 
• 
Liste des séances existantes (matière, classe, enseignant, date, heure) 
• 
Formulaire d'affectation : sélectionner enseignant + classe + matière + date + heure 
 
Interface ENSEIGNANT 
mes_seances_screen.dart 
• 
Récupère GET /enseignant/seances.php?id=X les séances de l'enseignant 
connecté 
• 
Affiche : matière, classe, date, heure dans des Cards 
• 
Bouton « Faire l'appel » sur chaque séance → navigue vers appel_screen 
 
appel_screen.dart 
• 
Reçoit la séance sélectionnée en paramètre du constructeur 
• 
Charge la liste des étudiants de la classe concernée 
• 
Affiche un CheckboxListTile par étudiant (coché = présent) 
• 
Bouton « Valider l'appel » → POST /enseignant/absences.php avec la liste 
complète 
• 
Affiche un SnackBar de confirmation après envoi réussi 
 
Interface ÉTUDIANT 
profil_screen.dart 
• 
Récupère GET /etudiant/profil.php?id=X les informations de l'étudiant connecté 
• 
Affiche : nom, prénom, email, classe dans une Card avec CircleAvatar
